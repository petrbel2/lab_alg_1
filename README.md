# lab_alg_1

Реализованные алгоритмы:

Алгоритм Гаусса без выбора ведущего элемента:
В цикле для каждой строки i матрицы кроме последней, из каждого элемента нижележащих строк j вычитается элемент i строки того же столбца домноженный на фактор(элемент[j][i] / элемент[i][i]). Из правых частей вычитается i-ная правая часть домноженная на фактор. Таким образом, все элементы i-тых столбцов строк ниже i-ной обнуляются. В результате возникает верхнетреугольная матрица.

Обратная подстановка позволяет вычислить x.

Алгоритм Гаусса с выбором ведущего элемента:
Сходен с первым алгоритмом, но перед каждой итерацией цикла i-ная строка заменяется на одну из еще не обработанных строк с наибольшим по модулю элементом i-ного стобца(также меняются местами правые части).

LU-разложение:
L - нижняя треугольная матрица с единицами на главной диагонали
U - верхняя треугольная матрица
Поскольку верхняя строчка L = {1, 0,..0), мы можем однозначно найти верхнюю строчку U - она равна верхней строчке A(это следует из перемножения матриц). Тогда каждый элемент самого левого стобца L равен соответствующему элементу левого столбца A, разделённому на левый верхний элемент L(остальной стобец заполнен 0). 
Теперь всего по одной неизвестной осталось в следующих строке U и столбце L, что позволяет найти их аналогично, и так далее.

Найдя LU, надо решить системы Ln = b u Ux = n. Вторая система эквивалентна изначальной LUx = b с обоими частями разделёнными на L, но в двух решаеммых системах содержится по одной треугольной матрице и они легко решаются прямой и обратной подстановками.


Функции в программе:

float** generate_matrix(int n) - генерирует случайную матрицу заданного размера

float** copy_matrix(int n, float **matrix) - копирует матрицу заданного размера

float* generate_b(int n) - генерирует случайную правую часть размера n

float* copy_b(int n, float*b) - копирует float массив размера n

void matrix_deleter(float**matrix_to_delete, int n) - удаляет двумерный массив заданного размера

float* Hauss_without_choice(int n, float **start_matrix, float *start_part) - решает СЛАУ простым методом Гаусса 

float* Hauss_with_choice(int n, float **start_matrix, float *start_part) - решает СЛАУ методом Гаусса с выбором ведущего элемента

std::pair<float**, float**> LU(int n, float**start_A) - выполняет LU разложение матрицы

float* direct_substitution(float** matrix, float* right_part, int n) - решение СЛАУ с нижнетреугольной матрицей прямой подстановкой

float* back_substitution(float** matrix, float* b, int n) - решение СЛАУ с верхнетреугольной матрицей обратной подстановкой

float** Hilbert_matrix_maker(int n) - генерирует матрицу Гильберта заданного размера

float* Hilbert_answers(float**matrix, int n) - считает правую часть для матрицы Гильберта если x = {1, 1...1}

float relative_inac(float* results, int n) - подсчет относительной ошибки для полученных X и X = {1, 1,...1}

double nevyaska(float**matrix, float*results, float*right_part, int n) - подсчёт невязки


Выводы:

1) Простой метод Гаусса - самый быстрый, метод Гаусса с выбором ведущего элемента незначительно медленнее, LU-разложение примерно в 2 раза медленнее. При этом почти все время затрачивается на собственно разложение, подстановки тратят в сотни раз меньше времени.

2) При вычислении ответов для одной матрицы A и множественных правых частей LU разложение наиболее эффективно, уже при 10 правых частях оно работает в разы быстрее

3) LU разложение накапливает меньше всего ошибок на матрице Гильберта. Метод Гаусса с выбором ведущего элемента накапливает значительно меньше ошибок, чем прпостой метод Гаусса, но на матрице Гильберта ошибки значительны с любым методом.
